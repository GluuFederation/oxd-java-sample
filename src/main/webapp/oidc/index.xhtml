Important terminology:

  • Client or Relying Party (RP): A client application requiring End-User Authentication and Claims from an OpenID Provider. In this case, this webapp is the client.
  • Authorization Server: The OpenID Provider (OP). For example, the Gluu Server instance you are pointing to by the "OP Host" field in the settings page.

Oxd-Java allows developers to implement the OpenID Connect Authorization Code Flow for authentication as well as to obtain basic profile information about the End-User.

The following is a typical flow:

  1. Client prepares an authentication Request.
  2. Client sends the request to the Authorization Server.
  3. Authorization Server authenticates the End-User and sends him back to the Client with an Authorization Code.
  4. Client sends a request for tokens using the Authorization Code.
  5. Client receives a response that contains an ID Token and Access Token.
  6. Client sends a request to obtain user claims using the Access Token.
  7. Client logs out user from the application and from the Authorization Server.
